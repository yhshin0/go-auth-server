# ADR-008: GET /auth/me — JWT 검증만 수행, Redis 세션 확인 생략

- **Status**: Accepted
- **Date**: 2026-03-15

---

## Context

`GET /auth/me`는 access token을 기반으로 현재 유저 정보를 반환하는 엔드포인트다.
Access token 검증 수준을 결정해야 했다.

- **JWT 서명 + exp 검증만**: 토큰 자체의 유효성만 확인.
- **JWT 검증 + Redis 세션 존재 확인**: 로그아웃 여부까지 실시간 확인.

---

## Decision

**JWT 서명 및 exp 검증만 수행한다. Redis 세션 조회는 하지 않는다.**

---

## Rationale

### Redis 세션 확인을 생략한 이유

Redis 세션까지 확인하면 로그아웃한 유저를 실시간으로 차단할 수 있다. 그러나 `/auth/me`는 단순 조회 엔드포인트이며, 매 요청마다 Redis 조회를 추가하는 비용 대비 실익이 크지 않다.

Access token의 TTL은 10분이다. 로그아웃 이후 최대 10분간 `/auth/me` 호출이 가능하지만, 이 엔드포인트는 유저 정보를 읽는 것 외에 부작용(side effect)이 없다.
10분의 공백은 이 용도에서 수용 가능한 수준이다.

Redis 세션 검증은 비용 대비 효과가 명확한 고위험 작업(비밀번호 변경, 결제, 권한 변경 등)에 적용하는 것이 적절하다.

### JWT 검증만으로 충분한 이유

JWT는 서버가 서명한 토큰이다. 서명 검증과 exp 확인만으로 토큰의 위변조 여부와 유효 기간을 신뢰할 수 있다. 단순 조회 API에서 이 수준의 검증은 표준적인 관행이다.

---

## Consequences

- 로그아웃 후 최대 10분(access token TTL) 동안 `/auth/me` 호출이 가능하다. 이는 의도된 트레이드오프다.
- 향후 `/auth/me`가 민감한 데이터를 반환하거나 쓰기 작업을 포함하게 되면 Redis 세션 검증 추가를 검토한다.
